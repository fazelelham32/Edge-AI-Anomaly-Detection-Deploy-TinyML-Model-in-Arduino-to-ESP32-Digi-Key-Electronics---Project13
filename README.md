# Edge AI Anomaly Detection with TinyML on ESP32

An end-to-end TinyML anomaly detection pipeline for monitoring electric motor vibration using an ESP32 microcontroller and a 3-axis accelerometer. This project demonstrates how vibration data can be collected, analyzed, modeled, converted, and deployed on embedded hardware using both traditional machine learning and deep learning techniques.

The implementation includes complete workflows for:

- Accelerometer-based vibration data acquisition
- Feature exploration and statistical analysis
- Training anomaly detection models
- TinyML model conversion
- Deployment on ESP32
- Optional inference on Raspberry Pi
- Real-time anomaly alert generation

---

# Project Overview

The project uses vibration signals captured from an electric motor to distinguish normal operating conditions from abnormal behavior.

Data acquisition is performed using:

- ESP32
- MSA301 3-axis accelerometer

Each vibration sample consists of approximately 200 measurements from all three accelerometer axes collected during one second.

The reference dataset was collected from a ceiling fan operating under multiple conditions, including:

- Fan Off
- Low Speed
- Medium Speed
- High Speed

Both normal and abnormal operating conditions are included.

Anomaly data was generated by attaching an additional weight to one fan blade to intentionally introduce mechanical imbalance.

# Machine Learning Approaches

Two independent anomaly detection methods are implemented.

## 1. Mahalanobis Distance

A statistical anomaly detector based on multivariate distance.

The model is trained using:

- Median Absolute Deviation (MAD)
- Mean vector
- Covariance matrix

The resulting statistical model is lightweight and suitable for embedded deployment.

Generated model format:

```
.npz
```

---

## 2. Autoencoder Neural Network

A deep learning model implemented with TensorFlow and Keras.

The autoencoder learns the distribution of normal operating behavior and identifies anomalies using reconstruction error (Mean Squared Error).

Generated model formats:

```
.h5
.tflite
C Header (.h)
```

The project notes that the Autoencoder provides significantly better performance when training data includes natural environmental variations.

# Data Collection

Data acquisition is performed using an ESP32 Feather connected to an MSA301 3-axis accelerometer. The ESP32 continuously captures vibration data and transmits the collected samples to a server over Wi-Fi.

The data collection firmware is located in:

```
data_collection/esp32_accel_post/
```

Before uploading the firmware, configure the following parameters inside the Arduino sketch:

- Wi-Fi SSID
- Wi-Fi Password
- Server IP Address

The collected samples are received and stored using:

```
data_collection/http_accel_server.py
```

The reference dataset contains vibration signals collected under multiple operating conditions, including both normal and abnormal motor behavior.

---

# Feature Analysis

Feature exploration is performed using the Jupyter Notebook located at:

```
data_collection/anomaly-detection-feature-analysis.ipynb
```

The notebook loads the collected datasets, generates feature visualizations, and allows comparison between different operating conditions to identify features capable of separating normal and anomalous behavior.

The dataset location and operating condition directories can be configured before executing the notebook.

# Model Training

## Mahalanobis Distance

The statistical anomaly detection model is trained using:

```
mahalanobis_distance/anomaly-detection-training-mahalanobis-distance.ipynb
```

Training uses normal and anomaly datasets to estimate:

- Mean Vector
- Covariance Matrix
- Median Absolute Deviation (MAD)

The trained model is exported as:

```
.npz
```

The generated models are stored in:

```
mahalanobis_distance/models/
```

---

## Autoencoder

The neural-network-based anomaly detector is trained using:

```
autoencoder/anomaly-detection-training-autoencoder.ipynb
```

The notebook trains an Autoencoder model using TensorFlow and Keras.

Training produces a model capable of identifying anomalies through reconstruction error (Mean Squared Error).

The trained model is exported as:

```
.h5
```

Generated models are stored inside:

```
autoencoder/models/
```

# Model Conversion

For Raspberry Pi deployment, the Mahalanobis Distance model can be used directly from the generated `.npz` file.

The Autoencoder model is converted from:

```
.h5
```

to

```
.tflite
```

using:

```
autoencoder/anomaly-detection-tflite-conversion.ipynb
```

For microcontroller deployment, both models are converted into constant C arrays stored in header files.

Mahalanobis conversion:

```
mahalanobis_distance/anomaly-detection-md-conversion.ipynb
```

Autoencoder conversion:

```
autoencoder/anomaly-detection-tflite-conversion.ipynb
```

The generated header files are copied into the corresponding Arduino deployment project.

# Custom Dataset Collection

The project supports collecting custom datasets using an ESP32 and MSA301 accelerometer.

The workflow consists of:

1. Uploading the data collection firmware.
2. Configuring Wi-Fi credentials and server address.
3. Mounting the accelerometer on the monitored motor.
4. Running the HTTP data collection server.
5. Recording samples for each operating condition.
6. Recording additional anomaly samples.
7. Using the collected dataset for feature analysis and model training.
